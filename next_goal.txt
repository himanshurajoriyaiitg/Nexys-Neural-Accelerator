FPGA-Based Tensor Processing Unit for Efficient
 Matrix Computation

Project purpose
- Multiply two signed N x N matrices on the Nexys A7-100T FPGA.
- Current target is N <= 8, with default N = 8.
- The FPGA stores matrices in BRAM, computes with a systolic MAC array, and returns results over UART.

High-level flow
1. A PC-side C program generates or reads matrix A and matrix B.
2. The C program sends those matrices to the FPGA over UART.
3. The board wrapper writes them into BRAM.
4. The controller starts the matrix-multiply engine.
5. The systolic array computes partial products and accumulated sums.
6. The FPGA stores matrix C in BRAM.
7. The FPGA returns matrix C over UART.
8. The C program saves A, B, and C into output files.
9. The Python checker recomputes A x B and compares it against the FPGA result.

FPGA resource intent
- BRAM:
  a_bram.v
  b_bram.v
  c_bram.v
- DSP:
  pe.v multiply path
- LUT and FF:
  controller.v
  uart_rx.v
  uart_tx.v
  nexys_a7_top.v
  counters, status registers, address generation, and control logic

Reset behavior
- Reset signal: rst_n
- Board input: CPU_RESETN
- Reset type: active-low asynchronous reset
- Effect:
  controller returns to IDLE from any state
  current run is aborted
  cycle counter clears
  UART state returns to idle
  status LEDs return to reset/idle indication

----------------------------------------------------------------------------
5. pe.v
----------------------------------------------------------------------------
Purpose
- One processing element in the systolic MAC array.

Working
- Multiplies one A value with one B value
- Adds the product into a local accumulator
- Forwards A horizontally and B vertically
- Uses DSP-friendly multiply coding style

Inputs
- clk
- rst_n
- clear_acc
- en
- a_in
- b_in

Outputs
- a_out
- b_out
- acc

----------------------------------------------------------------------------
6. systolic_array.v
----------------------------------------------------------------------------
Purpose
- Instantiates the full ARRAY_N x ARRAY_N PE mesh.

Working
- Connects PE outputs to neighboring PE inputs
- Feeds one A value per row and one B value per column
- Produces one accumulated output per PE

Inputs
- clk
- rst_n
- clear_acc
- en
- a_in[ARRAY_N]
- b_in[ARRAY_N]

Outputs
- c_out[ARRAY_N][ARRAY_N]

----------------------------------------------------------------------------
7. controller.v
----------------------------------------------------------------------------
Purpose
- Main control FSM for the matrix multiply engine.

Working
- Clears matrix C
- Loads A and B tile data
- Clears PE accumulators
- Runs the systolic array
- Writes results back to matrix C
- Advances tile_row, tile_col, and tile_k loops
- Raises done when all work is finished

States
- ST_IDLE
- ST_CLEAR_C
- ST_LOAD
- ST_CLEAR_ACC
- ST_RUN
- ST_WRITEBACK
- ST_DONE

Inputs
- clk
- rst_n
- start

Outputs
- clear_c_active
- load_active
- writeback_active
- clear_acc
- run_en
- busy
- done
- clear_c_addr
- tile_row
- tile_col
- tile_k
- load_count
- run_count
- wb_count

Reset detail
- Asynchronous active-low reset is implemented here
- When rst_n goes low, the controller returns to ST_IDLE from any state
- This immediately terminates the active operation

Outputs

- done

- c_host_rd_data

11. nexys_a7_top.v
----------------------------------------------------------------------------
Purpose
- Nexys A7 board-level top module.

Working
- Connects the board clock, reset, UART pins, and LEDs
- Parses UART command frames from the PC
- Writes matrix data into BRAM
- Starts the core
- Reads back matrix C and transmits it over UART
- Drives LEDs with cycle count and status signals

Inputs
- CLK100MHZ
- CPU_RESETN
- UART_TXD_IN

Outputs
- UART_RXD_OUT
- LED[15:0]

LED outputs
- LED[7:0]  : low 8 bits of cycle count
- LED[8]    : start latched
- LED[9]    : done latched
- LED[10]   : core busy
- LED[11]   : last command accepted
- LED[12]   : stream active
- LED[13]   : last command error
- LED[14]   : UART RX activity
- LED[15]   : UART TX busy

UART command interface
- 0x01: write A
- 0x02: write B
- 0x03: start
- 0x04: status
- 0x05: dump C

UART response interface
- 0x5A: ACK
- 0xE0: ERROR
- 0xA6: status packet
- 0xA7: dump packet

- matrix_b file
- matrix_c file

Outputs

----------------------------------------------------------------------------
14. uart_host.c
----------------------------------------------------------------------------
Purpose
- PC-side C host program for the real FPGA run.

Working
- Generates random matrices or reads input matrices from files
- Saves the exact matrices sent to the FPGA
- Sends matrix data over UART
- Starts the FPGA accelerator
- Polls FPGA status
- Reads matrix C back from the FPGA
- Saves the FPGA-returned output for later checking

Inputs
- --port
- --random or --matrix-a and --matrix-b
- --n
- --out-dir
- optional --seed, --baud, --timeout-ms

FPGA run output files
- fpga_output/matrix_a.txt
- fpga_output/matrix_b.txt
- fpga_output/matrix_c_fpga.txt
- fpga_output/run_info.txt

How to verify
1. Run the C host program.
2. Open matrix_a.txt and matrix_b.txt to inspect the exact sent inputs.
3. Open matrix_c_fpga.txt to inspect the exact returned FPGA result.
4. Run check_output.py using those three files.
5. PASS means the FPGA output matches the correct result for the exact inputs sent by the C program.
